Mapcache

Enables you to create, view, and share GeoPackages on your Android device.

GeoPackage MapCache has various tools and libraries which use their own implementation of GeoPackages. GeoPackage MapCache is an app for android and iOS that utilizes and demonstrates the functionality in GeoPackage. It downloads tiles from XYZ and WMS servers to use it in the disadvantage areas.

Create points and collect data in the field with custom forms and photo attachments. Geopackages created in desktop version or mobile version can be opened or shared to the mobile version of Mapcache application.

Offline Functionality: All the functionality used was online and can be accessed offline without any internet connection. It needs the map to load once prior to the loss of internet connectivity.

Steps:

Setting up -

  • Starting a new project by selecting ‘Create Project’.
  • Give appropriate name.

Functionality -

In the next step we used the Mapcache functionality options. We decided to choose range rings (to create rings by providing lat and long), markers, rectangles and polygons functionality options.

Save -

The next step is to Geopackage (It is the only option to save it). Then Import and open it in the android device.

Workflow

Pros and Cons

Benefits:

While having an open-source platform, it is completely free and does not require any licensing. The file (which was a layer) saved was in Geopackage format which does not consume much memory space.

Drawbacks and Limitations:

  • While the use of Geopackage is not a bad option but it is very limited and less user friendly and it was difficult to import the data or file into ArcGIS Pro.
  • It has limited data collection features which makes it difficult for the users to operate.
  • The application and the features available in it require high maintenance.

QField

Data collection app designed to work with QGIS

What is Q field? Q field is an app designed for mobile data collection in the field and integrates well with QGIS as it is based on QGIS with the same libraries and has basically an identical interface. The workflow by design has the user create/ configure their project in QGIS first then prepare and transfer their project/data for QField using a the QGIS extension QField Sync. They also have their own cloud-based service QCloud for synching data across devices.

Basic features include zoom, zoom to layer, listing features in a layer, temporal filter (time), access to logs, zoom to current location (based on GPS), undo button. Includes a lock feature which is useful for a still image display. It also includes a measuring tool and the ability to print to pdf.

Offline functionality: Any edits made offline will still save provided the information has been pre synched before. Must have data synched and loaded onto QField app prior to loss of internet connection. Alternatively, you can have data from the cloud downloaded onto your mobile device locally then synch back when internet connectivity returns. Any edits made can then be uploaded back to the cloud and accessed on QGIS.

Workflow

How to use QField

  1. Start new QFieldCloud Project on QGIS using existing project (or create one if needed)
  2. Synch with project data by logging into mobile QField App
  3. Make digitization and edits on mobile, save to cloud and push changes.
  4. On QGIS synched with cloud
  5. Edits made should appear in your QGIS project

The workflow depicted makes use QField and QField Cloud. It is possible to use only QField but that process involves manually transferring the QSynch prepared project and transferring it to your mobile device such as through USB or sharing options but this is not as efficient as using QField Cloud features.

Pros and Cons

Pros

  • Is open source so does not require licensing like Esri products
  • Has a very user-friendly UI that is intuitive and easy to navigate
  • Seamless integration into QGIS with an extension that is easy to use
  • QField Cloud is an easy way to sync data to your mobile device
  • QField Cloud allows you to download a copy of the project locally to your device in case of internet issues
  • Has a browse and edit mode
  • QGIS has the ability to use ArcGIS REST endpoints and display it in QGIS allowing you to use in QField
  • Free Q Field account has good amount of features
    • Unlimited public/private projects
    • Basic data hosting and versioning
    • File based projects only

Cons

  • Is open source so it is not part of Esri ArcGIS infrastructure
  • More QField Cloud features such as postGIS support, unlimited collaboration and better hosting and versioning will require a paid subscription
  • Personal experience: Had issues importing data from ArcGIS REST endpoint

Had an issue creating a link to ArcGIS REST endpoint for data where it would say the data layer is missing despite pointing to the REST endpoint. However this is more of a QGIS issue then a QField issue but is important to note as QField is designed to work with QGIS

ArcGIS FieldMaps

Esri's solution for offline mapping

Field Maps is Esri's main tool marketed for offline mapping. First we tried loading a web map with all 8 operational layers provided by the municipality of Clarington. Offline areas could not handle this much data. In order to fit the entire municipality in one offline area, we need to reduce the zoom extents. This reduces the usability of the data significantly.

Next we tried a smaller Web Map that only contained 3 operational layers was created. Field maps offline area was able to hold the entire extent of the municipality!

Success! But this only has 3 layers available offline, can we get more? Yes! Checkout the next section
