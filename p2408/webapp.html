Setting up a simple web application

SELECTING A TEMPLATE

The first step is to select from the numerous templates on experience builder. We decided to go with the blank full screen template for our solution here. This was so we could get a feel for the interface and operability from scratch.

CONNECTING TO THE DATA SOURCE(S)

The next step was to connect to our data. A web map of the municipality of Clarington we created via AGOL. Experience builders allow data in the form of feature services, map services, and web scenes. Other data types (GeoJSON, CSV, etc.) cannot be added as data sources.

ADDING WIDGETS TO THE WORKSPACE

After the data sources have been identified for the map, developers (with or without coding knowledge) can go ahead and add widgets to the map. Widgets can be used to achieve all sorts of results. From displaying map data to calculating spatial analysis. The data is consumed by widgets.

Drawbacks and Limitations:

We quickly discovered that the experience builder option did not support offline mapping or workflows for offline mapping in any form. We then tried to embed our offline map package we created for ESRI field maps via the Field Maps Designer, however, these offline areas cannot be viewed without the field maps mobile apps. The most you can share would be a bar code that can be scanned and a link. Both, however, only direct the end user to the ESRI Field mapsâ€™ download page, not the map itself. Right now, there is no way to pair Experience Builder and all its functionality with Field Mapsâ€™ offline functionality. As stated in the overview, the widgets do not have any proper dialogue for how they work and how they can be configured to reach their full potential. Requires a bit of repetitive trials and configuration to get it done.
